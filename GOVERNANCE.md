# Governance

## Project Model

UPP is maintained as an open, community-driven specification and reference repository.

## Maintainers

Maintainers are responsible for:

- Reviewing and merging pull requests
- Curating roadmap priorities
- Managing releases and security responses
- Preserving specification quality and interoperability

## Decision Making

- Small changes: Maintainer consensus in pull requests
- Material specification changes: Documented proposal + discussion in issues
- Breaking changes: Require versioning plan and migration notes

## Proposal Process

For significant changes:

1. Open a proposal issue with motivation and alternatives
2. Discuss impact and backward compatibility
3. Submit a PR with spec and example updates
4. Add migration notes when needed

## Release Policy

- Follow semantic versioning for specification milestones
- Publish changelog entries for each release
- Tag releases with clear compatibility notes

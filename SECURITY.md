# Security Policy

## Supported Versions

UPP is currently in early-stage development. Security fixes are applied on the default branch and included in the next tagged release.

## Reporting a Vulnerability

Please do not disclose vulnerabilities publicly before a fix is available.

To report a security issue:

- Open a private report to maintainers (preferred)
- Include detailed reproduction steps and impact
- Provide affected files/versions and any proof-of-concept

Maintainers will:

- Acknowledge receipt within 72 hours
- Validate and triage severity
- Coordinate a fix and disclosure timeline

## Disclosure Process

- Critical vulnerabilities are prioritized immediately
- A patch is prepared and reviewed
- A public advisory is published after remediation

## Scope

Security reports should focus on:

- Policy validation bypasses
- Rule-evaluation or enforcement integrity issues
- Supply-chain and dependency risks
- CI/CD or release tampering concerns
